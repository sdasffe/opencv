## 1. 引言

### 1.1 项目背景

为了将理论知识转化为实际编码能力，开发一个**图像处理小工具**，作为 **Qt + OpenCV** 学习的综合练手项目。通过本 Demo 实践以下技术点：

| 技术领域 | 实践内容 |
|---------|---------|
| Qt Graphics View | 图形视图、场景管理、自定义图形项 |
| Qt 信号槽 | 事件处理、参数变化驱动重算 |
| 自定义 QGraphicsItem | 可拖拽、可缩放的矩形/椭圆 ROI |
| 拖拽交互 | Drag & Drop 添加处理块 |
| OpenCV 图像处理 | 二值化、Otsu、形态学、滤波等 |
| C++ 架构设计 | 分层架构 + 策略模式（处理块可插拔） |

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 可视化调参 | 所有算法参数通过 UI 控件调节，实时预览效果 |
| ROI 标注 | 支持矩形、圆形等 ROI 区域标注，可拖拽缩放 |
| 链式处理 | 多个算法模块可串联，按序执行 |
| 可扩展 | 新增算法只需添加对应模块，不影响现有功能 |
| 性能可测 | 显示处理链总耗时，便于算法优化对比 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| ROI | Region of Interest，感兴趣区域 |
| 处理块 | 拖拽到参数面板中的单个算法配置单元（继承 `BaseBlock`） |
| 处理链 | 多个处理块按添加顺序依次作用于图像 |
| Otsu | 大津法，自动计算图像二值化阈值的算法 |

### 1.4 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 64 位 |
| Qt 版本 | Qt 6.7+（Widgets 模块） |
| 编译器 | MSVC 2019 64-bit（当前构建配置） |
| OpenCV 版本 | 4.12+（opencv_world 单库链接） |
| C++ 标准 | C++17 |
| 最低分辨率 | 1280×720 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 图片加载 | 10MB 以内图片加载 ≤ 1 秒 |
| 处理响应 | 1920×1080 单链处理 ≤ 200ms |
| 调参反馈 | 参数变化后 ≤ 100ms 刷新预览 |
| ROI 拖动 | 帧率 ≥ 30fps |
| 最大分辨率 | 支持 4096×4096 像素 |
| 内存占用 | 空闲 ≤ 100MB，处理中 ≤ 500MB |

### 3.2 易用性需求

- 参数调节即时反馈，无需点击确认
- 悬停 ROI 手柄时自动切换方向光标
- 所有算法提供合理默认参数（集中在 `AppConfig`）
- 操作异常弹出中文提示，不崩溃

### 3.3 可靠性需求

- 图片加载失败不影响程序继续运行
- 连续切换图片无明显内存增长
- 连续运行 2 小时不崩溃

### 3.4 可扩展性需求

- 新增算法：新增 `algorithms/` + `blocks/` 各一对文件，在 `createBlockByName()` 添加映射
- 所有处理块继承 `BaseBlock`，实现统一 `process()` 接口
- 算法逻辑与 UI 分离，算法层无 Qt Widget 依赖

### 3.5 可维护性需求

- 代码分层：表现层 → 应用层 → 领域层 → 基础设施层
- 公共接口和复杂逻辑添加中文注释
- 全局常量、默认值统一在 `config/appconfig.h` 管理

## 4. 系统架构

### 4.1 架构选型

本项目采用 **分层架构 + 策略模式**，共分 4 层：

```
┌─────────────────────────────────────────┐
│           Presentation Layer            │  表现层
│   core/widget.h/cpp（UI 交互、事件处理）   │
├─────────────────────────────────────────┤
│           Application Layer             │  应用层
│   core/imageprocessor.h/cpp（处理链调度）  │
├──────────────┬──────────────────────────┤
│ Domain Layer │  Infrastructure Layer    │  领域层 + 基础设施层
│ blocks/      │  utils/                  │
│ algorithms/  │  config/                 │
│ roi/         │                          │
└──────────────┴──────────────────────────┘
```

### 4.2 架构选型理由

| 候选方案 | 是否采用 | 原因 |
|---------|---------|------|
| 分层架构 | ✅ | 职责清晰，适合桌面应用长期维护 |
| 策略模式 | ✅ | 处理块可插拔，新增算法零侵入 |
| MVC/MVVM | ❌ | 项目规模小，分层已足够 |

### 4.3 模块职责

| 层 | 模块 | 职责 |
|----|------|------|
| 表现层 | `Widget` | 界面显示、ROI 管理、拖放、协调显示更新 |
| 应用层 | `ImageProcessor` | 处理链管理、ROI 坐标传递、耗时统计 |
| 领域层 | `blocks/` | 各处理块的 UI 与 `process()` 实现 |
| 领域层 | `algorithms/` | 纯 OpenCV 算法，无 UI 依赖 |
| 领域层 | `roi/` | 可交互 ROI 图形项 |
| 基础设施层 | `utils/` | QPixmap ↔ cv::Mat 转换、计时工具 |
| 基础设施层 | `config/` | 全局常量、默认值、样式色值 |

### 4.4 关键设计决策

| 决策 | 说明 |
|------|------|
| 原始图不可变 | `ImageProcessor` 始终从 `m_original` 开始执行处理链 |
| 信号驱动重算 | 参数变化 → `paramsChanged` → `reprocess()` |
| ROI 坐标映射 | Widget 负责场景坐标 → 图像像素坐标转换 |
| 处理块自管理 UI | 基类提供标题栏、使能、删除；子类填充参数区 |

### 6.2 模块依赖关系

```
widget ──→ imageprocessor ──→ blocks ──→ algorithms
   │                               │
   └──→ roi                         └──→ utils
                                          │
                                     config
```

**规则**：上层可依赖下层，下层不可依赖上层。

### 6.3 核心类说明

| 类名 | 职责 | 关键接口 |
|------|------|---------|
| `Widget` | UI 调度、ROI 管理、拖放 | `createBlockByName()`, `getCurrentRoiImageRect()` |
| `ImageProcessor` | 处理链执行、耗时统计 | `addBlock()`, `reprocess()`, `setRoiRect()` |
| `BaseBlock` | 处理块基类 | `process()`, `paramsChanged`, `removeRequested` |
| `BinarizationBlock` | 二值化 UI + 处理 | `process()` 支持全图/ROI 模式 |
| `ResizableRectItem` | 可缩放矩形 ROI | 8 手柄交互 |
| `ResizableEllipseItem` | 可缩放椭圆 ROI | 4 手柄交互 |
| `ImageConverter` | 格式转换 | `pixmapToMatRGB()`, `matToPixmap()` |
| `TimeMeasurer` | RAII 计时 | `elapsedMs()` |

### 6.4 新增算法步骤

以新增「边缘检测」为例：

1. 在 `algorithms/` 下新增 `canny.h/cpp`（纯 OpenCV 实现）
2. 在 `blocks/` 下新增 `cannyblock.h/cpp`（继承 `BaseBlock`，实现 UI 和 `process()`）
3. 在 `widget.cpp` 的 `createBlockByName()` 中添加名称映射
4. 在 `widget.ui` 算法列表中添加一项
5. 在 `opencv.pro` 的 SOURCES/HEADERS 中注册文件

**无需修改** `ImageProcessor`、`BaseBlock` 或其他已有处理块。

## 7. 数据设计

### 7.1 内部数据结构

| 数据 | 类型 | 持有者 | 说明 |
|------|------|--------|------|
| 原始图片 | `QPixmap` | `ImageProcessor::m_original` | 不可变，处理链始终从此开始 |
| 当前结果 | `QPixmap` | `ImageProcessor::m_result` | 处理链输出，用于显示 |
| ROI 矩形 | `QRectF` | `ImageProcessor::m_roi` | 图像像素坐标，空=全图 |
| ROI 图形项 | `ResizableRectItem*` / `ResizableEllipseItem*` | `Widget` | 当前仅单个 ROI |
| 处理块列表 | `QList<BaseBlock*>` | `ImageProcessor` + `Widget` | 按添加顺序执行 |

### 7.2 数据流

```
用户操作（调参 / 移动 ROI / 添加/删除处理块）
        │
        ▼
  paramsChanged / scene::changed / addBlock
        │
        ▼
  ImageProcessor::reprocess()
        │
        ├── 取 m_original
        ├── 遍历 m_blocks（跳过未启用块）
        │    Block1::process(input, roi) → 中间结果
        │    Block2::process(...) → ...
        └── TimeMeasurer 统计耗时
        │
        ▼
  emit processingFinished(elapsedMs)
        │
        ▼
  Widget 更新 GraphicsView + 耗时 Label
```

### 7.3 外部文件

| 文件类型 | 格式 | 用途 | 状态 |
|---------|------|------|------|
| 输入图片 | jpg/png/bmp/tiff | 待处理原始图片 | ✅ |
| 输出图片 | jpg/png | 保存处理结果 | ❌ 待开发 |
| 配置导出 | json | 保存处理链参数 | ❌ V1.2 规划 |

## 9. 风险评估与已知问题

### 9.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| OpenCV 与编译器不匹配 | 链接失败或运行时崩溃 | 确保 `opencv.pro` 中 lib 路径与编译器一致（当前 vc16 + MSVC） |
| 大图处理卡顿 | 调参体验差 | 引入防抖（debounce）、异步处理 |
| 单 ROI 限制 | 无法满足多区域对比 | V1.1 扩展为 `QList<QGraphicsItem*>` |
| 频繁重算 | ROI 拖动时 CPU 占用高 | `m_processingSuspended` 暂停机制（已有基础） |
