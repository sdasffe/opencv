## 3. 运行时协作（时序简图）

```mermaid
sequenceDiagram
    actor User as 用户
    participant W as Widget
    participant IP as ImageProcessor
    participant B as BaseBlock 子类
    participant Alg as *Algorithm

    User->>W: 打开图片 / 拖入算法 / 调参 / 拖 ROI
    W->>W: getCurrentRoiInfo()
    W->>IP: setRoi(roi)
    W->>IP: reprocess()
    loop 每个启用的块
        IP->>B: process(current, roi)
        B->>Alg: 纯算法计算
        Alg-->>B: cv::Mat
        B-->>IP: QPixmap
    end
    IP-->>W: processingFinished(ms)
    W->>W: 刷新画布显示结果
```

## 4. 读图要点

| 关系 | 含义 |
|------|------|
| `Widget → ImageProcessor` | 主窗口拥有引擎，负责同步 ROI 并触发 `reprocess` |
| `ImageProcessor ◇── BaseBlock*` | 处理链：按列表顺序调用 `process` |
| `BaseBlock ◄── 五个子类` | 策略模式：UI 参数 + `process` 实现 |
| `Widget → RoiInfo → 各 Block` | 图元几何 → 纯数据 → 算法 mask |
| `*Block ⋯> *Algorithm` | UI 与 OpenCV 算法分离 |

### 核心继承两条线

1. **处理块**：`QWidget` → `BaseBlock` → 五个具体算法块  
2. **ROI 图元**：`QGraphics*Item` → `Resizable*Item`  

中间用 `RoiInfo` + `ImageProcessor` 把 UI 和算法接起来。
